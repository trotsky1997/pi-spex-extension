Help me turn this rough idea into a spec-ready feature brief.

User input:

```text
$ARGUMENTS
```

Process:
- If the idea is underspecified, ask focused clarifying questions.
- Prioritize user value, scope boundaries, success criteria, and key edge cases.
- Keep the discussion concrete and avoid premature implementation details.
- End with:
  1. a concise problem statement
  2. a numbered list of clarified requirements
  3. open questions that still matter
  4. a final spec-ready prompt I can paste into `/speckit.specify`

If the idea is already clear, skip the extra questioning and go straight to the spec-ready brief.
